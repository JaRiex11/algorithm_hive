================================================================================
  РУКОВОДСТВО ПОЛЬЗОВАТЕЛЯ
  Программа эвристического размещения предметов по корзинам (алгоритм «пчёл»)
================================================================================

1. НАЗНАЧЕНИЕ ПРОГРАММЫ
-----------------------

Программа решает задачу упаковки: нужно распределить набор предметов
с заданными объёмами по корзинам ограниченной вместимости. Для поиска
решения используется эвристика, имитирующая поведение пчелиной колонии:

  - разведчики (sniffer bees) исследуют пространство решений;
  - медоносные пчёлы (honey bees) уточняют найденные варианты в окрестности
    лучших точек.

Программа запускается из консоли, параметры алгоритма и входные данные
задаются через текстовый интерфейс. Результаты можно выводить в консоль
и строить графики зависимости качества решения от параметров.


2. СБОРКА И ЗАПУСК
------------------

Сборка (MinGW, из каталога проекта):

  mingw32-make

Исполняемый файл:

  build\auto.exe

Запуск:

  build\auto.exe

Перед запуском в каталоге программы должен находиться файл input.txt
с описанием задачи (см. раздел 3).

При завершении работы программа ждёт закрытия всех окон с графиками,
если они были открыты.


3. ВХОДНЫЕ ДАННЫЕ (input.txt)
-----------------------------

Файл input.txt задаёт размер корзины и список предметов.

Формат:

  <вместимость_корзины> <количество_предметов>
  <объём_1> <объём_2> <объём_3> ... <объём_N>

Пример:

  100
  1000
  42 37 26 23 34 50 ...

  - первая строка: вместимость одной корзины (например, 100);
  - вторая строка: число предметов (например, 1000);
  - далее: объёмы всех предметов (целые числа), через пробел или с переносами.

Если файл не найден или формат неверный, программа выведет сообщение об ошибке.


4. ПАРАМЕТРЫ АЛГОРИТМА
----------------------

При старте используются значения по умолчанию:

  bee_honey_count     200    — число медоносных пчёл
  bee_sniffer_count    50    — число разведчиков
  distance              3    — «расстояние» между соседними решениями
  percent             0.2    — доля точек, участвующих в разведке (0..1)
  iteration           100    — число итераций алгоритма
  type_distribution uniform  — способ распределения пчёл по точкам

Параметры можно изменить командой s (см. раздел 6).


5. ГЛАВНОЕ МЕНЮ КОМАНД
----------------------

После запуска отображается приглашение «  >  ». Введите одну букву команды
и нажмите Enter.

  h — справка по командам (help)
  s — изменить параметры алгоритма (set)
  i — показать текущие параметры (info)
  r — выполнить один прогон алгоритма и вывести результат (run)
  g — серия прогонов с построением графика (graphics test)
  f — встроенный набор тестов с графиками (program test)
  q — выход из программы (quit)

Регистр буквы не важен: H и h — одна и та же команда.


6. ИЗМЕНЕНИЕ ПАРАМЕТРОВ (команда s)
-----------------------------------

После ввода s открывается подрежим настройки. Приглашение: «  >>  ».

Формат команды:

  <номер> = <значение>

Примеры:

  1 = 300
  4 = 0.15
  6 = 2

Список параметров:

  1 — bee_honey_count      число медоносных пчёл (целое)
  2 — bee_sniffer_count    число разведчиков (целое)
  3 — distance             расстояние между соседними точками (целое)
  4 — percent              доля точек для разведки (дробное, например 0.2)
  5 — iteration            число итераций (целое)
  6 — type_distribution    функция распределения пчёл (см. ниже)

Для параметра 6 укажите одну цифру сразу после «=»:

  6 = 1   — uniform  (равномерное распределение)
  6 = 2   — linear   (линейное)
  6 = 3   — square   (квадратичное)
  6 = 4   — exp      (экспоненциальное)

Дополнительно в подрежиме s:

  h — повторно показать справку по параметрам
  e или exit — вернуться в главное меню

Пустая строка игнорируется. Неверный формат выводит «bad command».


7. ПРОСМОТР ПАРАМЕТРОВ (команда i)
-----------------------------------

Выводит текущие значения всех параметров, перечисленных в разделе 4.


8. ОДИНОЧНЫЙ ПРОГОН (команда r)
-------------------------------

Запускает алгоритм с текущими параметрами на данных из input.txt.

В консоль выводится:
  - распределение предметов по корзинам;
  - загрузка корзин;
  - числовая оценка качества решения (quality).

Чем меньше значение quality, тем лучше решение (меньше корзин / меньше
перерасход вместимости — в зависимости от используемой метрики).


9. ТЕСТ С ГРАФИКОМ (команда g)
------------------------------

Позволяет перебрать один параметр в заданном диапазоне и построить график
зависимости quality от значения параметра.

После ввода g открывается подрежим. Приглашение: «  >>  ».

Формат команды:

  <номер> = <начало> <конец> <шаг>

Пример:

  4 = 10 50 10

Это означает: параметр №4 (iteration) от 10 до 50 с шагом 10.
Для каждого значения выполняется прогон алгоритма, в консоль печатается
качество, затем открывается окно графика.

Доступные параметры для теста:

  1 — bee_honey_count
  2 — bee_sniffer_count
  3 — distance
  4 — iteration

После завершения серии исходное значение изменённого параметра
восстанавливается.

Ось X графика — значение параметра, ось Y — quality.

  h      — справка в подрежиме теста
  e/exit — вернуться в главное меню

Шаг не должен быть равен 0.


10. ВСТРОЕННЫЕ ТЕСТЫ (команда f)
--------------------------------

Команда f запускает заранее заданный набор экспериментов без ручного ввода
диапазонов. Выполняются несколько серий прогонов; для каждой серии
строится отдельный график в окне «test».

Тесты включают (в порядке выполнения):

  1. Сравнение функций распределения (uniform, linear, square, exp)
     — по 5 прогонов на каждую функцию, усреднённое quality.

  2. Зависимость quality от соотношения числа медоносных пчёл и разведчиков
     (при фиксированной суммарной «популяции»).

  3. Зависимость quality от общего размера популяции пчёл.

Во время выполнения в консоль выводится прогресс (номер текущего шага).
По окончании нужно закрыть окна графиков, чтобы программа завершила работу
(если вы уже вышли из меню командой q).


11. ГРАФИКИ
-----------

Графики отображаются в отдельном окне (библиотека freeglut / easy_plot).

  - горизонтальная ось — значение варьируемого параметра;
  - вертикальная ось — quality;
  - подписи осей задаются автоматически (имя параметра и «quality»);
  - на осях отображаются числовые деления.

Закройте все окна графиков перед полным завершением программы.


12. ТИПИЧНЫЙ СЦЕНАРИЙ РАБОТЫ
----------------------------

  1. Подготовить input.txt с задачей.
  2. Запустить build\auto.exe.
  3. Команда i — проверить параметры.
  4. Команда s — при необходимости изменить, например:
       5 = 200
       1 = 250
  5. Команда r — получить решение в консоли.
  6. Команда g — исследовать влияние параметра, например:
       1 = 50 400 50
  7. Команда q — выход.


13. ЗАМЕЧАНИЯ
-------------

  - Результат стохастический: при одних и тех же параметрах quality может
    немного отличаться от запуска к запуску.
  - Файл input.txt читается при создании среды при старте программы;
    для смены задачи перезапустите программу с новым файлом.
  - Команда r не меняет параметры — только выполняет расчёт.
  - В подрежимах s и g для выхода используйте e или exit (проверяется
    первая буква строки).
